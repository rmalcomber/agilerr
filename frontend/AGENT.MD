# Frontend Agent Guide

This directory contains the main Agilerr product UI.

## Stack

- Preact
- Vite
- Tailwind CSS
- daisyUI
- PocketBase JS client for auth/session handling

## Responsibilities

- login and forced-password-change flows
- project dashboard, kanban, backlog, bugs, deleted, docs, API, MCP, users
- AI Add modal and review flow
- keyboard shortcuts and dense desktop-oriented app layout
- permission-aware action visibility

## Important Files

- `src/app.tsx`: primary application shell and most UI flows
- `src/lib/api.ts`: backend API client helpers
- `src/index.css`: product-level styles
- `tailwind.config.js`: theme/plugins
- `vite.config.ts`: dev proxy and local watcher config

## Commands

- Dev server: `npm run dev`
- Production build: `npm run build`
- Preview build: `npm run preview`

## Frontend Conventions

- Preserve the current dark visual language and dense information layout.
- Prefer route-aware behavior over large modal-only flows when users need context.
- Hide actions entirely when permissions do not allow them.
- Persist useful UI state in `localStorage` when it improves return-to-work flows.
- Keep descriptions, filters, and sidebars compact; the app targets serious project boards, not sparse marketing layouts.

## Notes

- The product frontend is built separately in dev mode but embedded into the backend for release.
- `backend/web/dist/` is regenerated by release builds; do not hand-edit embedded output.
